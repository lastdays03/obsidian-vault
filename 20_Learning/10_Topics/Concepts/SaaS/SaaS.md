# SaaS (Software as a Service)

## Definition
**SaaS**는 클라우드 기반으로 소프트웨어를 제공하는 모델로, 사용자는 별도의 설치 없이 구독(Subscription) 형태로 웹이나 앱을 통해 서비스를 이용합니다. 2026년 기준 전 세계 소프트웨어 시장의 50% 이상을 차지하며, **[[Vertical_SaaS]]**와 **AI Integration**이 핵심 트렌드입니다.

## Key Features & Pros/Cons
- **구독 모델**: 초기 투자 비용($0)으로 엔터프라이즈급 소프트웨어 사용 가능 (OpEx 중심).
- **자동 관리**: 보안 패치, 업데이트, 인프라 관리를 공급자가 전담.
- **확장성**: 사용자 수 증가에 따른 유연한 스케일링.
- **단점**: 데이터 주권 문제, 장기 구독 시 비용 누적, 벤더 종속성(Lock-in).

## SaaS vs [[PaaS]] vs IaaS (2026 Comparison)
| Feature      | SaaS (Software)               | [[PaaS]] (Platform)              | IaaS (Infrastructure) |
| ------------ | ----------------------------- | -------------------------------- | --------------------- |
| **Scope**    | Complete App                  | Dev Platform                     | Virtual Resources     |
| **Control**  | Minimal (Use only)            | App & Data                       | OS & Middleware       |
| **Examples** | Notion, Slack, Salesforce     | [[Vercel]], Heroku, [[Supabase]] | AWS EC2, GCP Compute  |
| **User**     | End User                      | Developer                        | SysAdmin / DevOps     |
| **Trend**    | AI-Native / [[Vertical_SaaS]] | AIaaS / Low-code                 | GPU-Specific Cloud    |

## 2026 Market Trends
1. **[[Vertical_SaaS]] Breakdown**: 특정 산업(의료, 법률, 건설 등)에 특화된 솔루션이 범용 SaaS(Horizontal)보다 높은 점유율을 보임.
2. **AI-First**: 단순 챗봇을 넘어 자율 AI 에이전트 기능을 내장한 SaaS가 표준이 됨.
3. **Usage-Based Pricing**: 단순 구독보다 사용량 기반 과금(Consumption-based) 모델로의 전환 가속화.
