# Knowledge Pipeline

## Definition
**Knowledge Pipeline**은 흩어진 정보(Data)를 체계적인 지식(Knowledge)으로 변환하고 유지 관리하는 일련의 워크플로우 흐름입니다.

## Stages
1.  **Capture (Inbox)**: 모든 아이디어와 메모를 수집합니다.
2.  **Review ([[Weekly Review]])**: 수집된 항목의 가치를 판단하고 분류합니다.
3.  **Harvest ([[Knowledge Harvester]])**: 가치 있는 노트를 영구 저장소(`20_Learning/10_Topics`)로 이관합니다.
4.  **Distill ([[Concept Distiller]])**: 노트에서 핵심 개념을 추출하여 원자적 지식(`20_Learning/00_Concepts`)으로 만듭니다.
5.  **Structure ([[MOC Builder]])**: 노트 간의 관계를 지도로 만들어 연결성을 강화합니다.
